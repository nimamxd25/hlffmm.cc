使用Hugo+github建立你的第一个博客

很多时候会发现事情做完了，但是却没有记录，想要回溯的时候就只剩下模糊的记忆了，所以建立个人博客就是一个很好的选择。

建站的主要过程分为三步：

  1. 安装hugo
  2. 生成网页，在本地对网页进行调试
  3. 将网页推送至GitHub

经过上述三步，即可生成一个可以访问的网页。以下对建站的全过程进行记录，希望对你也有帮助。

安装hugo

hugo在windows下的安装可以通过Chocolatey和Scoop安装，这里仅记录通过Scoop安装hugo的方法。

安装scoop

  1. 在powershell中通过以下命令安装

    1

    iex (new-object net.webclient).downloadstring('https://get.scoop.sh')

  2. 执行以下命令确认安装是否成功

    1

    scoop help

    若正常显示帮助信息则表示安装成功

通过scoop安装hugo

  1. 执行以下命令安装hugo

    1

    scoop install hugo

  2. 使用以下命令进行验证

    1

    hugo version

    正确显示以下信息则安装完成

创建 Hugo 网站

创建Blog

  1. 在电脑某个位置建立一个文件夹，用于存放网页文件，如e:/hugo-site
  2. 在powershell中输入cd e:/hugo-site进入文件夹
  3. 通过hugo new site blog建立网站，会生成一个blog文件夹，存放相应的网页文件

使用主题

刚刚生成的网站是没有任何内容的，需要下载个主题跑起来才有内容。Hugo 社区有很丰富的主题，可以通过官网 Themes 菜单选择自己喜欢的风格。主题下一般都会有很详细的安装及配置说明。

安装你的第一个主题

我选择的是Clean White这个主题，以下讲述一下配置过程。

  1. cd blog进入网页文件夹，通过以下命令将主题下载到theme文件夹下 git submodule add https://github.com/zhaohuabing/hugo-theme-cleanwhite.git themes/hugo-theme-cleanwhite
  2. 将themes/hugo-theme-cleanwhite/exampleSite文件夹下的所有文件，复制到blog目录下
  3. 通过hugo server命令，运行成功后，访问 http://localhost:1313 地址即可看到网页内容
  4. 通过hugo server命令本地调试网页，对blog文件夹内容的修改，是实时同步到 http://localhost:1313网页的

更换主题

如果想更换主题，可以重复上述步骤。也可以git submodule add下载好主题后，通过hugo serve -t 主题名来使用不同主题生成网页
